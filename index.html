Meilisearch - Local Full-Text Search Engine
▼
Meilisearch Index
Fast, open-source, locally-hosted search engine. Stores extracted text and metadata from all firm documents. Returns ranked results with relevance scores.
Open Source (Free)
Installation
Windows Service via NSSM
Bind Address
127.0.0.1:7700 (local only)
Index Location
C:\indexes\research
Re-index Schedule
Nightly at 2:00 AM (Task Scheduler)
📄 Document Records
One per file - contains filename, path, client, folder, summary, page count, modified date
📑 Chunk Records
Multiple per large file - contains extracted text, page ranges (pp. 44-49), parent reference
Norconex File System Crawler
Enterprise file system crawler - handles crawling, text extraction, and change detection. No custom code required for these functions.
Open Source (Free)
Runtime
Java (requires JRE)

MD5 checksum + metadata

OCR Support

4
MCP Server - AI Bridge (Query Intelligence)
▼
FastMCP Python Server
Exposes search tools to CoWork as callable functions. Translates natural language queries into structured search parameters using Claude before querying engines.
Open Source
🔍 search_work_files
Parameters: query (string), year (optional), category (optional), limit (default: 20)
Returns: Ranked document/chunk results from Meilisearch
📧 search_emails
Parameters: query (string), date_from (optional), date_to (optional), limit (default: 20)
Returns: Ranked email results from MailStore API
Query Intelligence Layer
Claude translates natural language or voice-transcribed queries into clean, structured search parameters. The user speaks naturally - Claude translates - search engines receive precise keywords.
📝 Query Translation
Extracts: core keywords (2-5 words), client filter, date/year filter, folder scope (Research/Clients/Email/All), search intent
🎤 Voice Input Handling
Strips filler words, normalizes spoken dates ("twenty twenty four" → "2024"), corrects homophones ("ten thirty one" → "1031")
🔄 Multi-Pass Search
Executes parallel searches for multi-topic queries, merges results, deduplicates by path, re-ranks by confidence
❓ Clarification Handling
Returns clarification questions for ambiguous queries instead of low-quality results
Confidence Scoring
Meilisearch scores (0.0-1.0) mapped to human-readable tiers:
High (0.85-1.00)
Strong topical match - directly relevant
Medium (0.60-0.84)
Related content - likely useful
Low (0.00-0.59)
Weak match - tangentially related
